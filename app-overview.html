App Overview

Trello Clone

A full-featured project management app inspired by Trello — built as a portfolio-grade full-stack application.

React 18 Node.js + Express PostgreSQL Prisma ORM JWT Auth Drag & Drop Playwright E2E

Product Perspective

Everything you need to manage work

From signing up to organizing complex projects — here's what users can do with the app.

🔐

Authentication

Users can register with a name, email and password, or log in to an existing account. Sessions are maintained with secure refresh tokens — users stay logged in across page reloads without re-entering credentials.

📋

Boards

Each user has their own personal workspace of boards. Create boards with a custom title and a color accent to visually distinguish projects. Boards are private — each user sees only their own.

📑

Lists

Within any board, create as many lists as needed (e.g. "To Do", "In Progress", "Done"). Lists can be reordered by dragging them left and right — order is persisted immediately to the server.

🃏

Cards

Add cards to any list to represent tasks. Cards can be dragged within a list to reorder them, or moved across lists to reflect progress. Position changes are saved in real time.

📝

Card Details

Click any card to open a detail modal. Add a rich description, set a due date, attach colored labels for categorization, and organize sub-tasks via checklists with individual items that can be checked off.

🏷️

Labels

Tag cards with labels — each with a custom text and a color. Labels are visible on cards at a glance in the board view, making it easy to scan and filter work by type or priority.

✅

Checklists

Break down complex cards into step-by-step checklists. Each checklist has a title and any number of items that can be individually checked. Progress is tracked visually as items are completed.

📅

Due Dates

Set a due date on any card. The app displays a smart due-date badge that changes appearance based on urgency — neutral for upcoming, amber for near-due, and red for overdue cards.

↕️

Drag & Drop

The full board is drag-and-drop enabled. Reorder lists horizontally and cards within or across lists — all with smooth animations. Position changes are auto-saved behind the scenes.

🔐 Auth Security Design

1

Registration / Login

Password hashed with bcrypt. Server returns a short-lived access token (JWT) and sets a long-lived refresh token in an httpOnly cookie.

POST /api/auth/register POST /api/auth/login
2

Authenticated Requests

The React app stores the access token in memory (React state — not localStorage) to prevent XSS. Every API call includes Authorization: Bearer <token>.

XSS-safe: not in localStorage
3

Token Refresh

When the access token expires, the client silently calls POST /api/auth/refresh. The httpOnly cookie is sent automatically — JavaScript can't read it, protecting against XSS.

httpOnly cookie — JS-inaccessible
4

Logout

Refresh token is cleared from the cookie server-side, invalidating the session.

POST /api/auth/logout

Tests Perspective

Three layers of test coverage

The project has server integration tests, client component tests, and full browser end-to-end tests — each with dedicated tooling.

5 Server integration test suites
5 Client component test suites
4 E2E browser test specs
🟩

Server — Integration Tests

Jest Supertest Real DB (test schema)

Tests hit a real test database via Supertest — no mocking. Each suite truncates all tables before running so tests are isolated and repeatable. Requires a separate .env.test database to protect dev data.

auth.test.js
Register (success, duplicate email, missing fields), login, token refresh, logout flows.
boards.test.js
Create, read, update, delete boards. Ownership isolation — users can't see each other's boards.
lists.test.js
Add lists to boards, reorder via position, delete — with authorization checks.
cards.test.js
Create cards in lists, move across lists, update positions, delete.
middleware.test.js
Auth middleware: rejects missing token, invalid token, expired token. Validates protect guard.
npm test (in /server)
⚛️

Client — Component Tests

Vitest React Testing Library jsdom

Component-level tests run in a jsdom environment with React Testing Library. They validate rendering behaviour, user interactions, and conditional UI logic — no browser required.

AuthContext.test.jsx
Tests auth context provider: initial state, login/logout state changes, token propagation.
BoardCard.test.jsx
Board card component renders title and color correctly; click handlers work.
BoardView.test.jsx
Board view renders lists and cards; shows empty state; drag context integration.
CardModal.test.jsx
Card detail modal renders description, labels, checklists; form interactions update state.
DueDateBadge.test.jsx
Badge displays correct styles for upcoming, near-due, and overdue dates.
npm test (in /client)
🎭

E2E — Browser Tests

Playwright TypeScript Full stack required

End-to-end tests spin up a real browser (Chromium) and drive the full application — frontend, API, and database all running. Tests use unique email addresses to avoid collisions between runs.

auth.spec.ts
Register, login, logout flows in browser. Validates redirect to /boards on success; error states on failure.
boards.spec.ts
Create a board, see it in the list, navigate to it, delete it. Validates board isolation between users.
lists-cards.spec.ts
Add lists and cards to a board. Rename them inline. Drag and drop to reorder. Verify persistence on reload.
card-detail.spec.ts
Open card modal, add description, set due date, add labels and checklists, check off items.
npx playwright test (in /e2e) Requires both servers running
